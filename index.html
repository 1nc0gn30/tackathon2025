Concept Overview

Agent Runners sit between people and the Netlify CLI. They translate prompts into vetted, reproducible operations with least-privilege access.

Developers define allow-lists, guardrails, and prompts. Clients trigger approved actions without ever touching raw CLI commands. Production sites stay safe, while workflows become programmable.

Programmable policies

Client vs Developer Roles

Client / Partner View

  • Trigger pre-approved deployments
  • Request AI summaries of incidents
  • Check production status without secrets

No shell access. Every click becomes an Agent-mediated prompt.

Developer / Admin View

  • Define allow-listed CLI verbs
  • Author AI prompts exposed to clients
  • Scope tokens to environments and sites

Developers turn the CLI into governed automation instead of ad-hoc terminals.

Workflow Guardrails

Agent Runners evaluate policies before touching the CLI. Clients see intent, not commands.

Allow: netlify status
Allow: netlify deploy --prod
Prompt: "Explain last deploy failure"

Result: Reduced client-side risk + faster developer response times.

Think: Netlify Dashboard → Add-Ons → Agent Runners. A new layer for governed automation atop hosting + deploys.

Narrative

Agent Runners convert natural language into safe Netlify CLI operations. Developers ship policies once; clients trigger them with confidence. Every action is auditable, replayable, and reversible through the Agent.

This prototype is a vision for a future Netlify Add-On: AI-mediated workflow guards that extend Netlify beyond deploy + host into governed automation.
